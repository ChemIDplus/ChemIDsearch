There are 2 user interfaces to help you learn to build valid ChemIDplus API Search URLs: this application and the Swagger UI. We expect this ChemIDplus API Search application will be more useful for most users, especially for building multi-expression searches.

To start generating Search URLs using this application, you will want to click on the "Create Search" button. After creating an expression and clicking "Done", you can create multi-expression searches by clicking the "+" button to add more.

For example, to search for all Analgesics, you would use one expression:

  • Category equals Analgesics

Whereas, to search for all Analgesics within the Drug Information Portal, you would use two expressions:

  • Category equals Analgesics
  • Locator equals DrugPortal

If you instead wanted to find all Analgesics not within the Drug Information Portal, you would check the "NOT" box on that expression:

  • Category equals Analgesics
  • NOT Locator equals DrugPortal

After entering the expression(s) and clicking "Get Results", the browser navigates to the HTML view of the results. To investigate the API Search URLs and JSON results for this search, click the "VIEW" button on the top right and select JSON. You will then see two API Search URL links and two JSON results. The top section is for Totals; the URL includes data=totals and thus only returns the count of matching substances.

Then the next section is for all the per-substance data types. The default data parameter is data=summary. You can also use the "Data Parameter Choice" button to select any of the other data parameters instead. The JSON results for the selected parameter are listed below that.

An alternative to using this ChemIDplus API Search is the Swagger UI. This includes more machine-readable information and is available at https://chem.nlm.nih.gov/api/swagger-ui.html. However, it does not include details about how to generate valid multi-expression searches.
