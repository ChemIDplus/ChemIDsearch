There are 2 user interfaces to help you learn to build valid ChemIDplus API Search URLs: this application and the Swagger UI. We expect this ChemIDplus API Search application will be more useful for most users, especially for building multi-expression searches.

Click the Help button above to read how to create and edit both single- and multi-expression searches.

After entering expression(s) and clicking the Get Results button, the browser navigates to the HTML view of the results. To investigate the API Search URLs and JSON results for any search, click the VIEW button on the top right and select "JSON". You will then see two API Search URL links and two JSON results. The top section is for Totals; the URL includes data=totals and thus only returns the count of matching substances.

Then the next section is for all the per-substance data types. The default data parameter is data=summary. You can also use the Data Parameter Choice button to select any of the other data parameters instead. The JSON results for the selected parameter are listed below that.

An alternative to using this ChemIDplus API Search is the Swagger UI. This includes more machine-readable information and is available at https://chem.nlm.nih.gov/api/swagger-ui.html. However, it does not include details about how to generate valid multi-expression searches.
