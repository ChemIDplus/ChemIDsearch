There are two user interfaces to help you learn to build valid ChemIDplus API Search URLs: this application and the Swagger UI. We expect this ChemIDplus API Search application will be more useful for most users, especially for building multi-expression searches.

Click the Help button above to read how to create and edit both single- and multi-expression searches.

After entering expression(s) and clicking the Get Results button, the browser navigates to the HTML view of the results. To investigate the API Search URLs and JSON/XML/TSV results for any search, click the View button on the top right and select "API". You will then see two API Search URL links and two JSON results. The top section is for Totals; the URL includes data=totals and thus only returns the count of matching substances. Then the next section is for all the per-substance data types.

The default data returned is the summary of each matching substance. You can use the Data button to return other information for each matching substance.

The default result format is JSON. Using the Format button, you may instead request XML or TSV. TSV is similar to CSV but uses the tab character as the delimiter; which means the output can be read by your favorite spreadsheet program.

The results based on the selected parameters are listed below those two choices.

An alternative to using this ChemIDplus API Search is the Swagger UI. This includes more machine-readable information and is available at https://chem.nlm.nih.gov/api/swagger-ui.html. However, it does not include details about how to generate valid multi-expression searches.
