When you don't need all data fields, pick a more focused data parameter. When you only request the data fields you need, you get back more results more quickly and have less to parse. The maximum batch size per data parameter is set to return similar average kilobytes per batch. Using the most specific data parameter for your needs results in higher maximum results per batch.

In the table below, the indented parameter names indicate a tree structure, where all indented data is included in the less-indented parent(s). Thus, "summary(2,000)" and "names(1,000)" are both also included in the results of "details(300)," which is itself included in the results for "complete(25)".
