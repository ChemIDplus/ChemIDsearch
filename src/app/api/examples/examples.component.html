Single expression searches include the expression in the URL path.
Thus the search "Category equals Analgesics" is https://chem.nlm.nih.gov/api/data/category/equals/analgesics?data=totals
The abbreviated form is https://chem.nlm.nih.gov/api/data/ca/eq/analgesics?data=totals
Except for searches on structure/smiles, the expression value is not case-sensitive.

Multi-expression searches have two or more expressions that must all be true to match substances. The expressions are listed in the query string. Each expression uses the key "exp" and the value is the same path as the single expression version except path slashes are replaced with %2F
The search "Category equals Analgesics AND Locator equals DrugPortal" is https://chem.nlm.nih.gov/api/data/search?data=totals&exp=category%2Fequals%2Fanalgesics&exp=locator%2Fequals%2Fdrugportal
The abbreviated form is https://chem.nlm.nih.gov/api/data/search?data=totals&exp=ca%2Feq%2Fanalgesics&exp=lo%2Feq%2Fdrugportal

When using a multi-expression search, expressions may also be negated, such that the search is of the form A AND NOT B. At least one expression must not be negated. To negate an expression, add "not" to the beginning of the operator.
Thus "Category equals Analgesics AND NOT Locator equals DrugPortal" is https://chem.nlm.nih.gov/api/data/search?data=totals&exp=ca%2Feq%2Fanalgesics&exp=lo%2Fnoteq%2Fdrugportal

The examples above all include data=totals. Any valid data parameter could be used with the same searches to return different levels of detail for all matching substances.
