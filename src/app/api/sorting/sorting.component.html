All API calls return sorted results. The default sort is usually by ascending RN/ID. The exception is Similarity searches, which by default are sorted first by descending Similarity, and second by ascending RN/ID.

You can specify a primary sort by appending "sortBy1=" followed by a sort field paramter value. If you want to reverse the default sort direction, also include "sortBy1Reverse=true". You may also request a secondary sort by including "sortBy2=" and, if you wish "sortBy2Reverse=true".

To search for analgesics, retrieve summary data, and sort by descending citations (the default sort direction for citations), use https://chem.nlm.nih.gov/api/data/category/equals/analgesics?data=summary&sortBy1=citations
To search for small molecules, retrieve summary data, and sort by descending weight (not the default sort direction for weight), plus add a secondary sort of descending citations, use https://chem.nlm.nih.gov/api/data/weight/between/1-5?data=summary&sortBy1=weight&sortBy1Reverse=true&sortBy2=citations

The system accepts any of the following sort fields:

Sort Field Parameter Value Default Direction Description
similarity DESC Only valid for similarity searches
lastMod DESC Last Modified
citations DESC Useful for relative ordering by PubMed search results; assume the actual number is stale
id ASC RN or System ID
name ASC Preferred Substance Name
mesh ASC MeSH
formula ASC  
weight ASC Calculated from the structure
has3d Yes first 3D images can be generated by most of our structures, but not all
inchikey ASC InChIKey
