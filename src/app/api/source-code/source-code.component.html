To begin consuming our API, we recommend using our source code to start with a functioning application.

This ChemIDsearch application works by consuming the same API you want to use. It has all the files needed to both create well-formed API requests and parse the JSON results. The resulting objects are strongly typed and the objects' properties have readable labels.

The source code and installation instructions are available at https://github.com/ChemIDplus/ChemIDsearch. The project is built on the Angular 2+ framework and coded in Typescript. For assistance with understanding how the components and classes work, please see the Angular documentation.

Some highlights of our code:

  • The expression.ts and search.ts files compose the API request URLs.
  • The server-json.ts file converts the JSON responses into our strongly typed classes.
  • The resource.ts file defines how to create complete resource/locator URLs by combining Source pre-URLs with Resource post-URLs. It also shows that {{ '{' }}RN} and {{ '{' }}MH} values in the pre-URL must be replaced with RN and MeSH values from the Summary.

Once you explore our code and have a basic understanding, you can start to customize it or extract what you need for your own purposes.
