On the Create an Expression page, the (automatic) field is selected by default. When you use it, the application returns the first match found in the following fields:

  1. RN
  2. Name
  3. ID
  4. UNII
  5. InChIKey
  6. Formula
  7. Category
  8. Locator
  9. Number

Therefore, the (automatic) field works best for RN, Name, ID, and UNII searches. If you are looking for an InChIKey, the first match might be a Name or UNII, and the application would return that instead.

To guarantee the results will be the field type you want, first select your desired field instead of using the (automatic) field.

Additionally, multi-expression searches may not contain any expression with the (automatic) field. Nor is the regex operator available when using the (automatic) field.
