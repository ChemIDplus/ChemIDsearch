Sometimes, only complex logic evaluating a particular value will provide the desired results: combining optional elements or filtering out undesired matches. In these cases, the slow but powerful regex operator can generate the perfect results.

However, regular expressions are difficult to get correct and should only be used by advanced users. Incorrectly formed expressions could drain our resources and prevent our servers from being able to respond as quickly to other users. We don't want to have to block any users due to undue strain, so please only use this operator if you are confident you can create good regular expressions.

Searches are case insensitive.

Terms will try to match characters anywhere in the value; use ^ and/or $ to specify matches must begin and/or end with the search term.

Literal special characters in chemical names that have meanings within regular expressions need to be escaped. In the examples below, when searching for the literal characters "+" and "[", they needed an "\" before them.

Note that when put into a URL, many special characters need to be escaped (again), so the URL terms often become incomprehensible! In addition, "\" and "/" need to be double escaped as "%255C" and "%252F" respectively.

Note that the regex operator may not be used with the (automatic) field.
